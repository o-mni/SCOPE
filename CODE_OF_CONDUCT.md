# Code of Conduct

## Our Commitment

SCOPE is a defensive security tool. The community around it should reflect the same values: professional, responsible, and focused on protection — not harm.

We are committed to making participation in this project a respectful, harassment-free experience for everyone, regardless of age, background, experience level, gender identity, nationality, religion, or any other personal characteristic.

---

## Our Standards

**We expect contributors and community members to:**

- Communicate professionally and respectfully in all project spaces
- Accept constructive feedback without hostility
- Focus discussion on technical merit and project goals
- Acknowledge when you are wrong, and learn from it
- Give credit to others for their ideas and work
- Respect that maintainers make final decisions about project direction

**We will not tolerate:**

- Harassment, intimidation, or personal attacks of any kind
- Discriminatory language or behavior
- Publishing others' private information without consent
- Threatening or violent language, even if framed as a joke
- Sustained disruption of discussions or contributions
- Deliberate attempts to introduce offensive, exploitative, or harmful code under the guise of contribution

### Project-Specific Standard: No Offensive Tooling

Because SCOPE is a security project, we hold an additional standard:

**Do not advocate for, introduce, or normalize offensive security techniques in this project's spaces.**

This includes:
- Suggesting that SCOPE should include exploit modules, attack automation, or evasion logic
- Sharing working exploits, payloads, or attack infrastructure in issues or PRs
- Pressuring maintainers to relax the defensive-only constraint

Discussion of offensive techniques in a clearly educational or defensive context (e.g., "this is how attackers do X, which is why we check Y") is welcome and encouraged. The line is advocacy for including offensive functionality in SCOPE itself.

---

## Scope of This Code of Conduct

This Code of Conduct applies to all project spaces including:

- GitHub Issues, Pull Requests, and Discussions
- Code review comments
- Project documentation contributions
- Any other communication channel linked from this repository

It also applies when an individual is representing the project in public — for example, at a conference, in a blog post, or on social media.

---

## Enforcement

### Reporting

If you experience or witness behavior that violates this Code of Conduct, report it to the project maintainer.

**GitHub:** Open a private report or contact the maintainer directly via GitHub profile contact information.

All reports will be treated with confidentiality. We will not disclose the identity of reporters without their consent.

### What Happens After a Report

1. The maintainer will acknowledge receipt of the report within 5 business days.
2. The situation will be reviewed and assessed.
3. A response will be determined — ranging from a private warning to a permanent ban from project spaces, depending on severity.
4. The outcome will be communicated to the reporter.

We do not publicly name individuals involved in enforcement actions unless the behavior was itself public and egregious.

### Consequences

Violations of this Code of Conduct may result in:

| Severity | Response |
|---|---|
| Minor, first offense | Private written warning with explanation |
| Repeated or significant | Temporary ban from contributing or commenting |
| Severe or repeated pattern | Permanent ban from all project spaces |
| Introduction of harmful code | Immediate PR rejection and ban |

---

## Philosophy

This project is maintained in the belief that good security tools make the internet safer for everyone — especially for organizations that cannot afford enterprise security teams. That mission only works if the community around the project is trustworthy.

We are not trying to police ideas. We are trying to build something useful and responsible together. Hold yourself to the standard you would want others to hold when they have access to your systems.
