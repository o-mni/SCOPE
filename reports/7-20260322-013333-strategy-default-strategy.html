Purpose & Objectives

This document defines the strategy, scope, and approach for the planned security assessment. It is intended to align stakeholders on objectives, methodology, and expected outcomes before the assessment begins.

  • Identify security misconfigurations and weaknesses in the target system.
  • Evaluate compliance against organisational security baseline.
  • Provide actionable remediation guidance for identified findings.
  • Establish a current security posture snapshot for tracking improvement over time.
2

Assessment Scope

In Scope

  • Operating system configuration and kernel parameters
  • Installed packages and service versions
  • Running services and open network ports
  • User accounts, groups, sudo rules, and SSH configuration
  • File system permissions and SUID/SGID binaries
  • Firewall and network filtering rules
  • Logging and audit configuration
  • Authentication and PAM configuration

Out of Scope

No explicit exclusions defined. All default SCOPE modules are in scope.

Constraints & Assumptions

  • Assessment will be conducted without root privileges unless explicitly granted.
  • No active exploitation or offensive techniques will be used.
  • All data collection is read-only and non-destructive.
  • System availability will not be impacted during assessment.
3

Methodology

SCOPE performs a structured, non-intrusive audit of local system state using OS-native interfaces and configuration file inspection. No network scanning or active exploitation is performed.

Assessment Phases

Phase 1 Reconnaissance & Data Collection

Collect system metadata, installed packages, running services, and configuration files using SCOPE's data collection modules.

Phase 2 Configuration Analysis

Evaluate collected data against security baseline rules, known misconfiguration patterns, and hardening benchmarks.

Phase 3 Finding Classification & Triage

Classify all findings by severity, assign remediation priority, and group by category for reporting.

Phase 4 Reporting & Handover

Generate the preliminary assessment report, review findings with stakeholders, and deliver remediation guidance.

Planned Modules

Module selection will be finalised prior to assessment execution. All default SCOPE modules are planned to run unless marked optional or excluded by scope constraints.
